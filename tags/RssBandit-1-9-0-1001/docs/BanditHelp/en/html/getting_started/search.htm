Subscribing to a lot of RSS feeds can start to feel like Drinking from an RSS firehose. It doesn’t take long to build up a huge list of subsribed feeds. Fortunately, RSS Bandit provides rich searching capabilities.

RSS Bandit has two search facilities built in: web search and local search. Local Search searches the contents of all your downloaded items and returns the results within a folder underneath the folder named Search Folders in the Feed Subscriptions tree.

Clicking on the Search button in the toolbar (see Figure 1) will bring up the search window (see Figure 2). This is a collapsible window so it can be collapsed within RSS Bandit to provide more detail display screen area. The window contains a textbox for entering the search phrase, a button to initiate the search, and several search options within collapsible panes below.


Figure 1: Search Button.


Figure 2: Docked Search Window.

By default, entering text into the textbox and clicking Search will perform a simple text search within the Description, and Title (if posts does not provide content) elements of every RSS Item.

However, starting with Rss Bandit version 1.5.*, RSS Bandit now makes use of the DotLucene search engine, a direct port of the Apache Lucene engine for Java.

Advanced Fields
The Advanced Fields option shown in figure 5 provides several options for tuning search results.

To enable an advanced option, click on the button beside the option name and the red bar will be converted to a green check mark.


Figure 5: Advanced Fields.

Read Status - enabling this option allows you to filter posts by their read status. If you enable the Read Status option and check the Unread Posts checkbox only unread items will be returned in the search. If the checkbox is unchecked then only read posts are returned. When this feature is disabled, both read and unread posts are returned in search results.

You can not define a search with Read Status specified only for internal technical reasons! To get only unread items, please use the new special folder named Unread Items.

There are three mutually exclusive ways to filter search results based on the date of the items.

  • Item Age - enabling this option lets you filter search results by the age of the feed item. The age is determined by the date when the item was downloaded into RSS Bandit. For example, you can check Older than and then select 2 hours in the dropdown to limit search results to feed items that were downloaded over two hours ago. Likewise, selecting Younger than and 1 day will limit search results to items that are less than a day old.
  • Item posted - enabling this option lets you filter items by the date they were posted. The post date is determined by the date the item was posted to the server as indicated by the RSS (or ATOM) feed. RSS indicates this via the pubDate element. The dropdown gives you the option to filter posts at, before, or after the date selected in the datepicker control.
  • Item posted between - enabling this option filters items based on a date range. This is similar to the Item posted option described above.

Search Scope
The scope for a search can be narrowed to a selection of categories or even individual feeds. Figure 8 shows the Search Scope option. It contains a tree view with a checkbox next to each category and feed. By selecting and unselecting feeds and categories, you can narrow or expand the scope of your search.
