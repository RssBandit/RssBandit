Advanced Configuration Options

General

Like other DOT.NET Framework application RSS Bandit supports configuration options specified in a file named RssBandit.exe.config. The location is always the path you installed the program, usually C:\Program Files\RssBandit and can be edited with a normal text editor like Notepad or your favourite XML editor.

This file defines e.g. options to allow execution with currently released CLR runtime version (<startup><supportedRuntime> entries), but also some specific options only valid for RSS Bandit. This options are all located within one section named <appSettings>.

The recommended way to change some of the options there is to create a new file with the name "RssBandit.exe.user.config" in the same directory. You should remind we will overwrite the original RssBandit.exe.config file each time you install an update of the application! As long you choose the same folder for uninstall/reinstall your newly created file will be left untouched but used with your customized options.

The contents of the file must look like this:

<appSettings>
...
</appSettings>

Each application option entry must be specified in this format:

<add key="[option key]" value="[option value]" />

So here is a more complete sample:

<appSettings>
<add key="MaxDownloadThreads" value="4" />
</appSettings>

Supported options

The following table describes all known and supported options with the allowed values.

Important note: the option keys must be spelled exactly as they are specified below!

 

Option key

Option values

Application Version

MaxDownloadThreads

Integer; default is 4; Recommended Range 1..16;

Amount of threads used to request feeds.

All

UnconditionalCommentRss

Boolean (true/false); default is false.

By default we consider the feed element tag <slash:comments> (namespace http://purl.org/rss/1.0/modules/slash/) containing the amount of comments posted to detect if we have to request the comment feed for a particular post. If the counter is zero, we will not request the comment feed, but if you set this option to true, we will always request the comment feed regardless the value of the counter.

v1.3.0.23 and higher

AutomaticColorSchemes

Boolean (true/false); default is true.

Set this to false if you consider any bad UI rendering effects while running software, that tweaks the windows default color scheme handling (Custom theme manager; Windows XP and higher) to provide "extended" schemes like the well known Window Blinds or StyleXP applications.

v1.3.0.24 and higher

UseEnglishReplyPrefix

Boolean (true/false); default is false.

If you reply to a post (feed must support this), a short string is used as a prefix similar to Outlook/Outlook express. If you really like to have English reply title prefix "Re: " instead of the localized (in German e.g. "AW: "), set this value to true.

v1.3.0.24 and higher

AppDataFolder

String (Operating System Path); default is <Documents and Settings>\<LoggedOnUser>\Application Data\RssBandit; this is usually part of Windows Roaming User Profile (RUP).

You can specify a custom path here like "D:\My Feeds\RSS Bandit". Do not forget to copy all the files from the original location and RSS Bandit should not be running while you do so!

v1.3.0.3 and higher

AppCacheFolder

String (Operating System Path); default is <Documents and Settings>\<LoggedOnUser>\Local Settings\ Application Data\RssBandit; this is usually not part of Windows Roaming User Profile (RUP).

You can specify a custom path here like "D:\My Feeds\RSS Bandit". It is allowed to specify the same path as used for setting AppDataFolder to have all related files in one location. Do not forget to copy all the files from the original location and RSS Bandit should not be running while you do so!

v1.3.0.42 and higher

Lucene.SearchIndexBehavior

String; the following values are supported (case sensitive):

  • NoIndexing - no search index is build/maintained, and local search feature is disabled
  • LocalAppDataDirectoryBased - Index is directory based and build relative to the user's local application data folder (default)
  • AppDataDirectoryBased - Index is directory based and build relative to the user's application data folder
  • TempDirectoryBased - Index is directory based and build relative to the user's temporary folder

v1.5.0.15 and higher

UseCookiesFromIE

Boolean (true/false); default is true.

By default we send cookies written by Internet Explorer (if they match the site) along with the feed request. Some feeds/hosts manage access to the feeds by informations stored in a cookie.

v1.3.0.39 and higher

RemoteFeedlist/ Ftp.ConnectionMode.Passive
(do not use spaces in the key name!)

Boolean (true/false); default is true.

Disable (false) this setting to configure the ACTIVE MODE for Synchronization using FTP.
