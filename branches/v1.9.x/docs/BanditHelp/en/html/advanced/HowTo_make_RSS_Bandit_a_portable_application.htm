How to: make RSS Bandit a portable application

Preface

Portable applications are running e.g. from a USB stick and are not require  installation on the target machine. The following sections describing the steps to apply to create a version of RSS Bandit executable from a stick. For the matter of sample we use PortableApps from www.portableapps.com - but other comparable stick software works similar. We assume, you have already a installation of the RSS Bandit on your machine where you setup the stick.

1. Copy files

  1. Step is: copy all files from the RSS Bandit installation folder (incl. all sub-directories) to a newly created folder directly below "PortableApps" folder located on your stick drive. That folder name does not matter, we suggest to use "PortableBandit".

  2. Step: create a new directory folder on the stick as the container for your subscriptions, flagged items, etc. - short: the RSS Bandit application data folder, e.g. "RssBanditData". Then open the RSS Bandit application data folder on your machine (located usually on Windows XP at C:\Documents and Settings\<YourUserName>\Application Data\RssBandit on Vista at C:\Users\<YourUserName>\Roaming\AppData\RssBandit. Copy all contained files (incl. sub-directories) to your newly created folder on the stick drive.

  3. Step: open the RSS Bandit local application data folder on your machine (located usually on Windows XP at C:\Documents and Settings\<YourUserName>\Local Settings\Application Data\RssBandit on Vista at C:\Users\<YourUserName>\Local\AppData\RssBandit. Copy the folder named "Cache" and the contained files to your newly created folder on the stick drive.

2. Changes required to configuration files

  1. Open the PortableApps\PortableBandit folder on your stick. Look for a file named RssBandit.exe.user.config. If it is not yet there, create that file using a simple text editor like Notepad. The minimal content required looks like this:

    <appSettings>
    ...
    </appSettings>  

    Read in help section Advanced Configuration Options about the syntax and other available options.

  2. Add the following (or modify the entries, if some of them already there):

    <add key="Lucene.SearchIndexBehavior" value="TempDirectoryBased" />
    <add     key="AppDataFolder" value="\RssBanditData" />
    <add     key="AppCacheFolder" value="\RssBanditData" />

    We assume, "F:" is the drive your stick was mounted to your machine. Please note the leading slash in the value attribute! This will put the folder to the root of the stick drive.

  3. In the same directory as above look for a file named RssBandit.exe.config and open it with Notepad. Scroll down near the end of the file and look for a section named <exceptionManagement ...>. Change the <publisher> file attribute value to get it point to your stick's application folder, e.g.:

    ... fileName="\RssBanditAppData\error.log" ...

    will create the error.log file in the stick's RssBanditAppData folder.

3. Changes required in Options

  1. Now run the PortableApps application from your stick and select "Options|Refresh App Icons" from the menu. The RSS Bandit application icon should now displayed.

  2. Start RSS Bandit from the stick.

  3. Open the Options dialog and change the following settings to point to a stick directory folder:

    1. Attachments/Podcasts Tab: change to download/podcast folders to an appropriate location on your stick.

    2. Web Browser Tab: if you configured to run a external executable with your visited Url's, deactivate that option or point to an application available on your stick as a portable application.
