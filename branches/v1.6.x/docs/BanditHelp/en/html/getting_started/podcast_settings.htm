Podcast Settings

One of the more exciting features in RSS Bandit 1.5 and above is its ability to now handle Podcasts. Let's walk through the various podcast settings now available.

The Podcast settings dialog can be accessed via the Tools Menu by clicking on Options (or hitting CTRL+O. Click on the Attachments/Podcasts tab.


Figure 1. The Attachments/Podcast Settings dialog.

Download Alert Window

Check the checkbox labeled Enable alert windows for newly downloaded attachments if you would like an alert window to display in the system tray.

Automatic Downloads

The next checkbox controls whether or not to automatically download podcasts.


Figure 2. Automatic Download Settings.

By default, when you subscribe to a new podcast, RSS Bandit will attempt to download every attachment still available in the feed (most feeds usually only display the last 10 or 15 items anyways).

However, you can configure RSS Bandit to only download the last few attachments by checking the checkbox labelled On initial subscription: only download last and then selecting the number of attachments in the dropdown.

Download Location

The next group of settings allow you to specify where to download podcast files.


Figure 3. Download Location Settings.

Checking the checkbox labeled Place downloads in subfolders named after the feed will create a subdirectory per feed. This is useful for keeping attachments organized and avoiding naming collisions.

Check the last checkbox Total dis psace used must not exceed to put a limito on how much disk space podcasts should take up. If that much space is used up, no more podcasts will be downloaded until you free up space.

Advanced Options

Click on the Advanced Options... button to bring up the advanced options dialog.


Figure 4. Advanced Options.

In general, any type of file can be an attachment to an RSS feed, such as a word doc. However, RSS Bandit lets you specify which file types should be treated in a special manner as a Podcast.

You can see the list of file types treated as podcasts by default in figure 4 above. Underneath the list of filetypes are options to copy the podcast file to a Windows Media Player playlist, an iTunes playlist, or a directory of your choice.

The second half of the dialog lets you specify Playlist options. For example, you can choose to use the RSS feed's name as a playlist so that each podcast feed has its own playlist (default) or you can add every podcast file to a single playlist.
